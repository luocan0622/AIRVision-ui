# AIRVision 自动化测试项目说明

## 1. 项目简介

本项目是针对 **AIRVision** Windows 桌面应用的 UI 自动化测试框架，基于 Python 生态构建，采用 **Page Object Model（页面对象模式）** 组织代码，使用 **pytest** 驱动测试执行。

**核心目标：**

- 模拟真实用户操作（鼠标点击、键盘输入、文件对话框交互）
- 覆盖 Projects / Workflows 等核心业务场景
- 支持单功能调试与完整流程回归

**技术栈：**

| 类别 | 技术 |
|------|------|
| 测试框架 | pytest |
| Windows 自动化 | pywinauto（UIA 后端）、pyautogui |
| 配置 | YAML |
| 日志 | loguru |
| 报告 | pytest-html、Allure（可选） |

**运行环境：**

- Windows 10 / 11
- Python 3.8+
- AIRVision 应用可正常启动

## 3. 架构设计

### 3.1 分层结构

```
测试用例 (tests/)
    ↓ 调用
页面对象 (pages/main_page.py)
    ↓ 继承
BasePage + FileDialogMixin
    ↓ 使用
公共工具 (utils/)
    ↓ 驱动
AIRVision 桌面应用
```

### 3.2 页面对象（POM）

| 类 | 职责 |
|----|------|
| `BasePage` | 元素查找、鼠标/键盘操作、弹出菜单、图像定位 |
| `FileDialogMixin` | Windows 文件对话框：地址栏导航、文件名输入、覆盖确认、Success OK |
| `MainPage` | AIRVision 主窗口业务：Projects、Workflows、工具栏 |
| `DialogTitle` | 各对话框窗口标题注册表，便于统一维护 |

### 3.3 交互原则

所有 UI 操作尽量**模拟真实用户**：

- 以**鼠标点击**为主要交互方式
- 文件对话框地址栏优先使用 **Alt+D + 粘贴 + Enter**
- 键盘输入优先使用 **pywinauto SendKeys**（比 pyautogui 更可靠）
- 文件名框使用 **Alt+N** 聚焦后键入

## 4. 配置说明

配置文件：`config/config.yaml`

### 4.1 应用配置（app）

```yaml
app:
  path: "C:/path/to/AIRVisionApp.exe"   # 被测应用路径
  title: "MainWindow"                  # 主窗口标题/类名
  backend: "uia"                         # uia（推荐）或 win32
  timeout: 10                            # 元素等待超时（秒）
  close_on_finish: false                 # 测试结束后是否关闭应用
```

> 调试阶段建议 `close_on_finish: false`，便于观察测试后的应用状态。

### 4.2 测试数据路径（test_paths）

```yaml
test_paths:
  project: "C:/Users/.../test/new_project"       # 新建/打开项目目录
  save_as_project: "C:/Users/.../Save_AS_Project" # 另存为目录
  template_images: "C:/Users/.../模型/重命名"     # 模板图片目录
  default_template_image: "moban1.png"
  default_depth_image: "moban2.png"
  default_workflow_filename: "Untitled-1.json"
```

### 4.3 环境配置（environment）

```yaml
environment:
  screenshot_on_failure: true
  screenshot_dir: "screenshots"
  log_dir: "logs"
  report_dir: "reports"
```

## 5. 命名规范

### 5.1 测试项目命名

| 场景 | 格式 | 示例 |
|------|------|------|
| 新建项目（文件名框输入） | `test_数字`（**不含**扩展名） | `test_1`、`test_6` |
| 磁盘上的项目文件 | `test_数字.airvision` | `test_1.airvision` |
| 项目数据目录 | `test_数字_data` | `test_1_data` |
| 工作流文件 | `test_数字.json` 或 `Untitled-数字.json` | `test_1.json` |

**自动编号规则**（`next_test_project_name()`）：

1. 扫描 `test_paths.project` 目录
2. 统计已有 `test_N.airvision` 和 `test_N_data` 占用的序号
3. 取最大序号 + 1，生成下一个名称

### 5.2 测试用例命名

- pytest 要求测试方法以 `test_` 开头
- 示例：`test_new_project`、`test_open_project`

## 6. 核心业务流程

### 6.1 新建项目（new_project）

```
Projects → New Project
  ① 地址栏 Alt+D 跳转到 PROJECT_PATH
  ② 文件名框输入 test_数字（不含 .airvision）
  ③ 点击「保存」
  ④ 覆盖确认（若有）→ 点击「是」
  ⑤ Success 提示框 → 点击「OK」
```

**代码示例：**

```python
page = MainPage(app)
project_name = page.new_project()          # 自动命名 test_N
project_name = page.new_project(filename="test_1")  # 指定名称
```

### 6.2 打开项目（open_project）

```
Projects → Open Project
  ① 地址栏跳转到 PROJECT_PATH（与新建相同）
  ② 文件名框（auto_id=1148）输入 test_N.airvision
  ③ 点击「打开」
```

**代码示例：**

```python
page.open_project()                              # 自动选取最新 test_N.airvision
page.open_project(filename="test_1.airvision") # 指定文件
```

### 6.3 设置默认模板图片（set_default_template_image）

> **前提：需已打开项目**

```
Projects → Set Default Template Image...
  ① Browse Template → 选择 moban1.png
  ② Browse Depth   → 选择 moban2.png
  ③ 点击 OK
```

### 6.4 工作流操作

| 方法 | 菜单路径 | 说明 |
|------|----------|------|
| `new_workflow()` | Workflows → New | 新建 Untitled-N |
| `rename_workflow()` | Workflows → Rename | 重命名为 test_N |
| `save_workflow()` | Workflows → Save | 保存到 `{项目}_data/workflows/` |
| `open_workflow()` | Workflows → Open | 打开 .json 工作流 |
| `close_workflow()` | Workflows → Close | 关闭当前工作流 |

工作流保存目录规则：

```
{项目目录}/{项目名}_data/workflows/
```

示例：`C:\...\new_project\test_1_data\workflows\test_1.json`

## 7. 测试体系

项目包含三类测试，适用于不同场景：

| 文件 | 类型 | 适用场景 |
|------|------|----------|
| `test_main_page.py` | **单步测试** | 调试单个功能（如只测新建项目） |
| `test_full_page.py` | **流程化测试** | Projects / Workflows 菜单串联 |
| `test_full_workflow.py` | **完整 E2E** | 10 步项目+工作流闭环 |

### 7.1 单步测试（test_main_page.py）

每个用例独立运行，互不影响：

```bash
pytest tests/test_main_page.py::TestProjectsMenu::test_new_project -v
pytest tests/test_main_page.py::TestProjectsMenu::test_open_project -v
pytest tests/test_main_page.py::TestProjectsMenu::test_set_default_template_image -v
```

### 7.2 流程化测试（test_full_page.py）

**Projects 全流程（6 步）：**

```
清理 → 新建 → 设置模板 → 保存 → 关闭 → 打开 → 关闭
```

**Workflows 全流程（4 步）：**

```
新建项目(前置) → 新建 workflow → 重命名 → 保存 → 关闭
```

```bash
pytest tests/test_full_page.py::TestProjectsFullFlow -v
pytest tests/test_full_page.py::TestWorkflowFullFlow -v
```

### 7.3 完整 E2E（test_full_workflow.py）

**10 步完整流程：**

1. 新建项目
2. 设置默认模板图片
3. 新建 workflow
4. 重命名 workflow
5. 保存 workflow
6. 保存项目
7. 关闭项目
8. 打开项目
9. 打开 workflow
10. 再次关闭项目

```bash
pytest tests/test_full_workflow.py -v -m slow
```

### 7.4 测试标记

| 标记 | 含义 |
|------|------|
| `@pytest.mark.smoke` | 冒烟测试，核心功能快速验证 |
| `@pytest.mark.regression` | 回归测试 |
| `@pytest.mark.slow` | 耗时较长的流程测试 |
| `@pytest.mark.integration` | 集成 / 流程测试 |

## 8. 快速开始

### 8.1 安装

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

### 8.2 环境检查

```bash
python check_env.py
```

检查项：Python 版本、依赖包、配置文件、应用路径、必要目录。

### 8.3 修改配置

编辑 `config/config.yaml`，至少修改：

- `app.path`：AIRVision 可执行文件路径
- `test_paths.project`：测试项目存放目录（需提前创建）

### 8.4 运行测试

```bash
# 全部测试
python run_tests.py
# 或
pytest

# 冒烟测试
python run_smoke.py
# 或
pytest -m smoke

# 生成 HTML 报告
pytest --html=reports/report.html --self-contained-html
```

## 11. 扩展开发指南

### 11.1 新增页面对象

1. 在 `pages/` 下创建文件，继承 `BasePage`
2. 用 Inspect.exe 获取控件 `AutomationId`
3. 定义定位器常量与操作方法

```python
from pages.base_page import BasePage

class SettingsPage(BasePage):
    BTN_APPLY = {"auto_id": "...", "control_type": "Button"}

    def click_apply(self):
        self.click(**self.BTN_APPLY)
```

### 11.2 新增对话框

在 `pages/dialogs.py` 的 `DialogTitle` 中注册窗口标题，然后在 `FileDialogMixin` 或 `MainPage` 中实现处理方法。

### 11.3 新增测试用例

```python
import pytest
from pages.main_page import MainPage

class TestMyFeature:
    @pytest.fixture(autouse=True)
    def setup(self, app):
        self.page = MainPage(app)

    @pytest.mark.regression
    def test_my_feature(self):
        self.page.new_project()
        # 断言...
```

### 11.4 新增流程化测试

参考 `tests/test_full_page.py`，在一个 `test_*` 方法内按顺序调用多个 page 方法，步骤间用 `logger.info` 标注进度。

## 12. 常见问题

### Q1：应用无法启动

- 确认 `config.yaml` 中 `app.path` 正确
- 手动双击 exe 确认可正常打开
- 运行 `python check_env.py` 排查

### Q2：元素定位失败

- 使用 `python inspect_controls.py` 查看控件树
- 确认窗口已获得焦点
- 增大 `app.timeout` 配置

### Q3：文件对话框地址栏输入失败

- 框架已使用 Alt+D + SendKeys + Enter 方案
- 查看日志中 `地址栏 Alt+D 输入` 相关条目
- 确认 `test_paths.project` 目录存在

### Q4：新建项目后 Success 框未关闭

- 框架在 `handle_new_project_dialog` 步骤 5 自动点击 OK
- 日志应出现：`[Success] 鼠标点击 OK`
- 若标题不是 `Success`，需在 `dialogs.py` 中更新 `PROJECT_CREATED_SUCCESS`

### Q5：set_default_template_image 失败

- **必须先打开或新建项目**，该菜单需要项目处于打开状态
- 确认 `test_paths.template_images` 目录存在且包含对应图片

### Q6：测试方法找不到（pytest ERROR: not found）

- pytest 只识别以 `test_` 开头的方法名
- 正确：`def test_open_project(self)`
- 错误：`def open_project(self)`

## 13. 应用生命周期

`AppManager`（`utils/app_manager.py`）负责：

| 行为 | 说明 |
|------|------|
| `start_or_connect()` | 应用已运行则连接，否则启动 |
| `is_running()` | 检测进程与主窗口是否存在 |
| `close()` | 优雅关闭，失败则强制结束进程 |

测试 session 级别由 `tests/conftest.py` 的 `app_manager` fixture 管理，避免每个用例重复启动应用。
