# AIRVision 自动化测试项目说明

## 1. 项目简介

本项目是针对 **AIRVision** Windows 桌面应用的 UI 自动化测试框架，基于 Python 生态构建，采用 **Page Object Model（页面对象模式）** 组织代码，使用 **pytest** 驱动测试执行。

**核心目标：**

- 模拟真实用户操作（鼠标点击、键盘输入、文件对话框交互）
- 覆盖 Projects / Workflows 等核心业务场景
- 支持单功能调试与完整流程回归

**技术栈：**

| 类别 | 技术 |
|------|------|
| 测试框架 | pytest |
| Windows 自动化 | pywinauto（UIA 后端）、pyautogui |
| 图像处理 | Pillow（工作流画布节点检测） |
| 配置 | YAML |
| 日志 | loguru |
| 报告 | pytest-html（`reports/report.html`）+ Airtest 风格步骤报告（`reports/airtest/`） |

**运行环境：**

- Windows 10 / 11
- Python 3.8+
- AIRVision 应用可正常启动

## 3. 架构设计

### 3.1 分层结构

```
测试用例 (tests/)
    ↓ 调用
页面对象 (pages/main_page.py)
    ↓ 组合
pages/mixins/ + pages/actions/ + BasePage + FileDialogMixin
    ↓ 使用
公共工具 (utils/) + 步骤报告 (step_reporter / airtest_report)
    ↓ 驱动
AIRVision 桌面应用
```

**推荐导入方式：**

```python
# 页面对象与公共类型
from pages import MainPage, WorkflowLayoutEngine, WorkflowBuildResult

# 测试辅助
from tests.support.helpers import add_tool_and_assert, preview_pipeline_on_canvas
```

**WorkflowMixin 继承链（工作流画布相关）：**

```
WorkflowMixin
  └── WorkflowPipelineMixin（链式放置 + 连线）
        ├── WorkflowBuilderMixin（build_detection_workflow 等高层 API）
        └── FilterMenuMixin（右键 Filter 菜单添加工具）
              └── WorkflowCanvasMixin（画布定位与节点断言）
```

布局计算由 `WorkflowLayoutEngine`（`workflow_layout.py`）独立完成，可在不启动应用的情况下单元测试。

### 3.2 页面对象（POM）

| 类 | 职责 |
|----|------|
| `BasePage` | 元素查找、鼠标/键盘操作、弹出菜单、图像定位 |
| `FileDialogMixin` | Windows 文件对话框：地址栏导航、文件名输入、覆盖确认、Success OK |
| `MainPage` | AIRVision 主窗口：组合 `pages/mixins/` 各业务 Mixin |
| `ProjectMixin` / `WorkflowMixin` 等 | 菜单与业务流程（位于 `pages/mixins/`） |
| `FilterMenuMixin` | 画布右键 Filter 菜单：搜索、滚动、添加工具节点 |
| `WorkflowPipelineMixin` | 多工具链式放置、顺序连线、`build_workflow_pipeline()` |
| `WorkflowBuilderMixin` | `build_detection_workflow()`、`preview_pipeline_layout()` 等高层 API |
| `WorkflowLayoutEngine` | 视口链式布局、Exec 端口坐标、缩放适配（纯计算） |
| `WorkflowPipelineDef` | 预定义检测流程（`workflow_pipelines.py` 注册表） |
| `PipelineRunResult` | 低层流水线结果：`plan` + `connections` |
| `WorkflowBuildResult` | 高层检测流水线结果：`plan` + `connections` + `pipeline` |
| `DialogTitle` | 各对话框窗口标题注册表，便于统一维护 |

### 3.3 共享数据类型（workflow_types.py）

| 类型 | 字段 | 说明 |
|------|------|------|
| `PipelineRunResult` | `plan`, `connections` | `build_workflow_pipeline()` 返回值；`plan.placements` 为**实际屏幕坐标** |
| `WorkflowBuildResult` | `plan`, `connections`, `pipeline` | `build_detection_workflow()` 等高层 API 返回值 |

两者均提供 `node_count` 属性（等于 `plan.node_count`）。

### 3.4 领域命名空间（pages/actions/）

`MainPage` 在构造时挂载三个命名空间属性，通过委托调用底层 Mixin 方法，**与直接调用 Mixin 方法完全等价**：

| 命名空间 | 类 | 常用方法 | 等价 Mixin 方法 |
|----------|-----|----------|-----------------|
| `page.projects` | `ProjectActions` | `new_project()` | `page.new_project()` |
| | | `open_project()` | `page.open_project()` |
| | | `ensure_open()` | `page.ensure_project_open()` |
| | | `get_file_path()` | `page.get_project_file_path()` |
| `page.workflows` | `WorkflowActions` | `new()` / `save()` / `close()` | `new_workflow()` 等 |
| | | `build_detection(name)` | `build_detection_workflow(name)` |
| | | `build_circle_workpiece()` | `build_circle_workpiece_detection()` |
| | | `build_extended_detection()` | `build_extended_detection_pipeline()` |
| `page.canvas` | `CanvasActions` | `add_tool(key)` | `add_workflow_tool(key)` |
| | | `has_tool_node(key)` | `has_workflow_tool_node_for_key(key)` |
| | | `preview_layout(keys)` | `preview_pipeline_layout(keys)` |

**代码示例：**

```python
page = MainPage(app)

# 传统写法（完全支持，现有测试均使用此方式）
page.new_project()
page.build_circle_workpiece_detection()

# 命名空间写法（推荐新代码使用，语义更清晰）
page.projects.new_project()
page.workflows.build_circle_workpiece()
page.canvas.add_tool("d3_circle_detection")
```

### 3.5 交互原则

所有 UI 操作尽量**模拟真实用户**：

- 以**鼠标点击**为主要交互方式
- 文件对话框地址栏优先使用 **Alt+D + 粘贴 + Enter**
- 键盘输入优先使用 **pywinauto SendKeys**（比 pyautogui 更可靠）
- 文件名框使用 **Alt+N** 聚焦后键入

## 4. 配置说明

配置文件：`config/config.yaml`

### 4.1 应用配置（app）

```yaml
app:
  path: "C:/path/to/AIRVisionApp.exe"   # 被测应用路径
  title: "MainWindow"                  # 主窗口标题/类名
  backend: "uia"                         # uia（推荐）或 win32
  timeout: 10                            # 元素等待超时（秒）
  close_on_finish: false                 # 测试结束后是否关闭应用
```

> 调试阶段建议 `close_on_finish: false`，便于观察测试后的应用状态。

### 4.2 测试数据路径（test_paths）

```yaml
test_paths:
  project: "C:/Users/.../test/new_project"       # 新建/打开项目目录
  save_as_project: "C:/Users/.../Save_AS_Project" # 另存为目录
  template_images: "C:/Users/.../模型/重命名"     # 模板图片目录
  default_template_image: "moban1.png"
  default_depth_image: "moban2.png"
  default_workflow_filename: "Untitled-1.json"
```

> 日志目录（`logs/`）、截图目录（`screenshots/`）、报告目录（`reports/`）由代码与 `pytest.ini` 固定，无需在配置文件中重复声明。

## 5. 命名规范

### 5.1 测试项目命名

| 场景 | 格式 | 示例 |
|------|------|------|
| 新建项目（文件名框输入） | `test_数字`（**不含**扩展名） | `test_1`、`test_6` |
| 磁盘上的项目文件 | `test_数字.airvision` | `test_1.airvision` |
| 项目数据目录 | `test_数字_data` | `test_1_data` |
| 工作流文件 | `test_数字.json` 或 `Untitled-数字.json` | `test_1.json` |

**自动编号规则**（`next_test_project_name()`）：

1. 扫描 `test_paths.project` 目录
2. 统计已有 `test_N.airvision` 和 `test_N_data` 占用的序号
3. 取最大序号 + 1，生成下一个名称

### 5.2 测试用例命名

- pytest 要求测试方法以 `test_` 开头
- 示例：`test_new_project`、`test_open_project`

## 6. 核心业务流程

### 6.1 新建项目（new_project）

```
Projects → New Project
  ① 地址栏 Alt+D 跳转到 PROJECT_PATH
  ② 文件名框输入 test_数字（不含 .airvision）
  ③ 点击「保存」
  ④ 覆盖确认（若有）→ 点击「是」
  ⑤ Success 提示框 → 点击「OK」
```

**代码示例：**

```python
page = MainPage(app)
project_name = page.new_project()          # 自动命名 test_N
project_name = page.new_project(filename="test_1")  # 指定名称
```

### 6.2 打开项目（open_project）

```
Projects → Open Project
  ① 地址栏跳转到 PROJECT_PATH（与新建相同）
  ② 文件名框（auto_id=1148）输入 test_N.airvision
  ③ 点击「打开」
```

**代码示例：**

```python
page.open_project()                              # 自动选取最新 test_N.airvision
page.open_project(filename="test_1.airvision") # 指定文件
```

### 6.3 设置默认模板图片（set_default_template_image）

> **前提：需已打开项目**

```
Projects → Set Default Template Image...
  ① Browse Template → 选择 moban1.png
  ② Browse Depth   → 选择 moban2.png
  ③ 点击 OK
```

### 6.4 工作流操作

| 方法 | 菜单路径 | 说明 |
|------|----------|------|
| `new_workflow()` | Workflows → New | 新建 Untitled-N |
| `rename_workflow()` | Workflows → Rename | 重命名为 test_N |
| `save_workflow()` | Workflows → Save | 保存到 `{项目}_data/workflows/` |
| `open_workflow()` | Workflows → Open | 打开 .json 工作流 |
| `close_workflow()` | Workflows → Close | 关闭当前工作流 |

工作流保存目录规则：

```
{项目目录}/{项目名}_data/workflows/
```

示例：`C:\...\new_project\test_1_data\workflows\test_1.json`

### 6.5 工作流流水线搭建（VisionMaster 式）

框架支持在画布上按预定义工具链自动布局、放置节点，可选顺序连线：

| 方法 / 对象 | 返回类型 | 说明 |
|-------------|----------|------|
| `build_workflow_pipeline(tool_keys, ...)` | `PipelineRunResult` | 通用：新建工作流 → 适配缩放 → 链式放置 → 可选连线 |
| `build_detection_workflow(pipeline, ...)` | `WorkflowBuildResult` | 按注册名或 `WorkflowPipelineDef` 搭建检测流程 |
| `build_circle_workpiece_detection(...)` | `WorkflowBuildResult` | 快捷：圆形工件 4 工具链 |
| `build_extended_detection_pipeline(...)` | `WorkflowBuildResult` | 快捷：扩展 8 工具检测链 |
| `preview_pipeline_layout(tool_keys, ...)` | `dict` | 仅计算布局预览（缩放、步长），不操作 UI |
| `add_workflow_tool(tool_key)` | `FilterToolDef` | 单个 Filter 工具添加 |
| `has_workflow_tool_node_for_key(tool_key)` | `bool` | 断言画布节点是否出现 |
| `finish_task()` | — | 任务结束后点击 OK 或等待（替代内部 `_after_task()`） |

**预定义流程**（`pages/mixins/workflow_pipelines.py`）：

| 名称 | 工具数 | 说明 |
|------|--------|------|
| `circle_workpiece_detection` | 4 | 相机 → 3D圆检测 → 结果条件 → 通讯发送 |
| `circle_workpiece_with_roi` | 5 | 含 ROI 裁剪的圆形工件检测 |
| `extended_detection` | 8 | 相机 → ROI → 圆检测 → 平面拟合 → 高度 → 结果条件 → 点云可视化 → 通讯 |

**代码示例：**

```python
page = MainPage(app)
fs.prepare_workflow_canvas(page)  # 确保项目与工作流标签已打开

# 搭建圆形工件检测流程（返回 WorkflowBuildResult）
result = page.build_circle_workpiece_detection(verify_nodes=True)
assert result.node_count == 4
assert result.plan.placements  # 实际屏幕坐标

# 低层 API：自定义工具链
run = page.build_workflow_pipeline(
    ("camera_data_source", "d3_circle_detection"),
    connect=False,
)
assert run.plan.v_gap > 0

# 或使用注册名
result = page.build_detection_workflow("circle_workpiece_detection")

# 命名空间写法
result = page.workflows.build_circle_workpiece(verify_nodes=True)

# 仅预览布局（单元测试可直接用 WorkflowLayoutEngine，无需启动应用）
from pages import WorkflowLayoutEngine
preview = page.preview_pipeline_layout(
    ("camera_data_source", "d3_circle_detection"),
    canvas_width=1209,
    canvas_height=600,
)
```

## 7. 测试体系

项目包含多类测试，适用于不同场景：

| 文件 | 类型 | 适用场景 |
|------|------|----------|
| `test_main_page.py` | **单步测试** | 菜单栏、语言切换、Projects、画布 Filter 工具 |
| `test_full_page.py` | **流程化测试** | Projects / Workflows 菜单串联 |
| `test_full_workflow.py` | **完整 E2E** | 15 步项目+工作流闭环 |
| `test_workflow_layout.py` | **单元测试** | 布局引擎纯计算（无需启动应用） |
| `test_filter_tools_demo.py` | **Demo 测试** | Filter 多种选取策略、随机批量添加 |
| `test_circle_workpiece_workflow.py` | **流水线 Demo** | 圆形工件 / 扩展 8 工具检测链 |
| `test_sample.py` | **示例** | 基础连通性、控件检查 |

### 7.1 单步测试（test_main_page.py）

每个用例独立运行，互不影响。主要测试类：

| 测试类 | 覆盖内容 |
|--------|----------|
| `TestMainPageMenuBar` | 顶部菜单栏可见性与点击 |
| `TestLanguageSwitch` | 中英文切换（含 `@pytest.mark.smoke`） |
| `TestProjectsMenu` | 新建项目、设置默认模板图片 |
| `TestWorkflowCanvasTools` | 画布右键 Filter 添加 3D 圆检测等工具 |

```bash
pytest tests/test_main_page.py::TestProjectsMenu::test_new_project -v
pytest tests/test_main_page.py::TestLanguageSwitch::test_language_button_visible -v
pytest tests/test_main_page.py::TestWorkflowCanvasTools::test_add_d3_circle_detection_tool -v
```

### 7.2 流程化测试（test_full_page.py）

**Projects 全流程（6 步）：**

```
清理 → 新建 → 设置模板 → 保存 → 关闭 → 打开 → 关闭
```

**Workflows 全流程（4 步）：**

```
新建项目(前置) → 新建 workflow → 重命名 → 保存 → 关闭
```

```bash
pytest tests/test_full_page.py::TestProjectsFullFlow -v
pytest tests/test_full_page.py::TestWorkflowFullFlow -v
```

### 7.3 完整 E2E（test_full_workflow.py）

**15 步完整流程：**

1. 检查应用已打开
2. 新建项目
3. 设置默认模板图片
4. 新建 workflow
5. 重命名 workflow
6. Filter 添加工具
7. 保存 workflow
8. 保存项目
9. 另存为项目
10. 关闭项目
11. 打开原始项目
12. 打开已保存的 workflow
13. 再次保存 workflow
14. 关闭 workflow
15. 再次关闭项目

```bash
pytest tests/test_full_workflow.py -v -m slow
```

流程步骤由 `tests/flow_steps.py` 统一封装（`run_step` / `step_*`），与 `test_full_page.py` 共用。每步通过 `execute_step()` 自动截图并写入 Airtest 报告。

### 7.4 测试辅助模块（tests/support/）

可复用的断言与布局预览逻辑，避免各测试文件重复编写：

| 函数 | 说明 |
|------|------|
| `add_tool_and_assert(page, tool_key, ...)` | 添加 Filter 工具并断言画布节点出现 |
| `preview_pipeline_on_canvas(page, tool_keys, ...)` | 读取画布尺寸并预览流水线布局，返回 `(preview, rect)` |

```python
from tests.support.helpers import add_tool_and_assert, preview_pipeline_on_canvas

fs.prepare_workflow_canvas(page)
```

## 10. 日志、截图与报告

### 10.1 日志

- 控制台：INFO 级别

### 10.3 Airtest 风格步骤报告

每个测试用例执行时，`StepReporter`（`utils/step_reporter.py`）自动记录每一步：

- 步骤标题、action 名称、耗时
- 每步操作后截图（复制到报告目录）
- 测试结束生成单用例 HTML 报告
- 会话结束生成汇总索引页

**输出目录结构：**

```
reports/airtest/
└── 20260611_142516/              # 会话时间戳
    ├── index.html                # 本次运行汇总索引
    └── tests_test_full_workflow.py_.../
        ├── report.html           # 单用例步骤报告
        └── screenshots/
            ├── step_001_检查_AIRVision.png
            └── ...
```

在流程测试中使用 `flow_steps.run_step()` 或 `execute_step()` 包装的步骤会自动纳入报告。纯单元测试（如 `test_workflow_layout.py`）同样会生成报告，但无 UI 截图。

## 11. 扩展开发指南

### 11.1 新增页面对象

1. 在 `pages/` 下创建文件，继承 `BasePage`
2. 用 Inspect.exe 获取控件 `AutomationId`
3. 定义定位器常量与操作方法

```python
from pages.base_page import BasePage

class SettingsPage(BasePage):
    BTN_APPLY = {"auto_id": "...", "control_type": "Button"}

    def click_apply(self):
        self.click(**self.BTN_APPLY)
```

### 11.2 新增对话框

在 `pages/dialogs.py` 的 `DialogTitle` 中注册窗口标题，然后在 `FileDialogMixin` 或 `MainPage` 中实现处理方法。

### 11.3 新增测试用例

```python
import pytest
from tests.support.helpers import add_tool_and_assert

class TestMyFeature:
    @pytest.fixture(autouse=True)
    def setup(self, page):
        self.page = page

    @pytest.mark.regression
    def test_my_feature(self):
        self.page.projects.new_project()  # 或使用 self.page.new_project()
        add_tool_and_assert(self.page, "d3_circle_detection")
```

### 11.4 新增流程化测试

参考 `tests/test_full_page.py` 与 `tests/flow_steps.py`：单步操作用 `flow_steps.step_*()` 或 `run_step()` 封装日志、截图与断言，在一个 `test_*` 方法内按顺序调用。

**`flow_steps.py` 主要步骤函数：**

| 函数 | 说明 |
|------|------|
| `run_step(title, action, func, ...)` | 通用单步包装（日志 + 截图 + 报告） |
| `prepare_workflow_canvas(page)` | 确保项目与工作流标签已打开 |
| `step_new_project` / `step_open_project` | 新建 / 打开项目 |
| `step_set_default_template` | 设置默认模板图片 |
| `step_new_workflow` / `step_rename_workflow` | 新建 / 重命名工作流 |
| `step_add_filter_tool` | 添加单个 Filter 工具 |
| `step_save_workflow` / `step_open_workflow` / `step_close_workflow` | 工作流存取 |
| `step_save_project` / `step_save_project_as` / `step_close_project` | 项目存取 |
| `step_build_circle_workpiece_pipeline` | 搭建圆形工件检测流水线 |
| `step_build_workflow_pipeline` | 通用流水线搭建（返回 `PipelineRunResult`） |

### 11.5 新增检测流水线

1. 在 `pages/mixins/workflow_pipelines.py` 定义 `WorkflowPipelineDef` 并注册到 `PIPELINE_REGISTRY`
2. 如需快捷方法，在 `WorkflowBuilderMixin` 中添加包装（参考 `build_circle_workpiece_detection()`）
3. 可选：在 `pages/actions/workflows.py` 添加命名空间别名
4. 在 `tests/flow_steps.py` 添加对应 `step_build_xxx()`（可选）
5. 编写测试：先 `prepare_workflow_canvas()`，再调用 `build_detection_workflow("your_pipeline_name")`

### 11.6 扩展领域命名空间

新增业务域时，在 `pages/actions/` 下创建类并继承 `PageActionsBase`，然后在 `main_page.py` 的 `__init__` 中挂载：

```python
# pages/actions/my_domain.py
class MyDomainActions(PageActionsBase):
    def do_something(self, *args, **kwargs):
        return self._call("do_something", *args, **kwargs)

# pages/main_page.py
self.my_domain = MyDomainActions(self)
```

## 12. 常见问题

### Q1：应用无法启动

- 确认 `config.yaml` 中 `app.path` 正确
- 手动双击 exe 确认可正常打开
- 运行 `python check_env.py` 排查

### Q2：元素定位失败

- 使用 `python inspect_controls.py` 查看控件树
- 确认窗口已获得焦点
- 增大 `app.timeout` 配置

### Q3：文件对话框地址栏输入失败

- 框架已使用 Alt+D + SendKeys + Enter 方案
- 查看日志中 `地址栏 Alt+D 输入` 相关条目
- 确认 `test_paths.project` 目录存在

### Q4：新建项目后 Success 框未关闭

- 框架在 `handle_new_project_dialog` 步骤 5 自动点击 OK
- 日志应出现：`[Success] 鼠标点击 OK`
- 若标题不是 `Success`，需在 `dialogs.py` 中更新 `PROJECT_CREATED_SUCCESS`

### Q5：set_default_template_image 失败

- **必须先打开或新建项目**，该菜单需要项目处于打开状态
- 确认 `test_paths.template_images` 目录存在且包含对应图片

### Q6：测试方法找不到（pytest ERROR: not found）

- pytest 只识别以 `test_` 开头的方法名
- 正确：`def test_open_project(self)`
- 错误：`def open_project(self)`

## 13. 应用生命周期

`AppManager`（`utils/app_manager.py`）负责：

| 行为 | 说明 |
|------|------|
| `start_or_connect()` | 应用已运行则连接，否则启动 |
| `is_running()` | 检测进程与主窗口是否存在 |
| `close()` | 优雅关闭，失败则强制结束进程 |

测试 session 级别由 `tests/conftest.py` 的 `app_manager` fixture 管理，避免每个用例重复启动应用。

每个测试自动启用 `_step_reporter_context`（autouse），初始化步骤记录器并在用例结束时生成 Airtest 报告。

常用 function 级 fixture：

| Fixture | 说明 |
|---------|------|
| `config` | 会话级配置（`config.yaml`） |
| `app_manager` | 应用启动/连接/关闭管理 |
| `app` | pywinauto Application |
| `page` | `MainPage(app)` |
| `page_esc` | `page` + 测试后按 ESC |
| `clean_page` | `prepare_clean_state()` 后再返回 `page` |
| `clean_page_esc` | 清理环境 + ESC 收尾 |

任务收尾请使用公开方法 `page.finish_task()`，勿在测试中直接调用 `_after_task()`。
